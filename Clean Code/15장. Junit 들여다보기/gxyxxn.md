우리는 처음 왔을 때 보다 더 깨끗하게 해놓고 떠나야 한다.

## 코드 개선

- 멤버 변수 앞에 붙인 접두어 제거
: 변수 이름에 범위를 명시할 필요가 없음
- 의도를 명확히 표현하기 위해 조건문 캡슐화
- 멤버 변수와 지역 변수 구분
-  부정문은 긍정문보다 이해하기 어렵기 때문에 반전.
- 변수 네이밍
- 코드 가독성
