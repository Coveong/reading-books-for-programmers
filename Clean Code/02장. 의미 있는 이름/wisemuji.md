# 의미 있는 이름을 짓기 위해선

우리는 변수나 클래스, 인수, 패키지 등 소프트웨어의 다양한 요소에 이름을 붙인다. 좋은 이름을 지으려면 시간이 걸리지만 좋은 이름을 절약하는 시간이 훨씬 더 많다. 이 장에서는 이름을 잘 짓는 간단한 규칙들을 소개한다.

### 의도를 분명히 밝히기

```kotlin
int d; // 경과 시간(단위: 날짜)
```

변수명 d에는 아무 의미도 드러나지 않는다. 경과 시간이나 날짜라는 느낌도 들지 않는다. 

```kotlin
int elapsedTimeInDays;
```

의도가 드러나는 이름을 사용하면 코드 이해와 변경이 쉬워진다. 한눈에 코드가 하는 일을 짐작할 수 있게 된다.

### 그릇된 정보를 피하라

- 코드에 그릇된 정보를 담는 것을 피해라.
- 나름대로 널리 쓰이는 의미가 있는 단어를 다른 의미로 사용해도 안된다.
- 서로 흡사한 이름을 사용하지 않도록 주의한다.
- 일관성이 떨어지는 표기법을 사용하지 않는다.

### 의미 있게 구분하라

예를 들어, 동일한 범위내에서는 다른 두 개념이 같은 이름을 사용하지 못한다. 그렇다고 해서 한쪽 이름을 마음대로 바꾸어서는 안된다. (예: 연속된 숫자를 덧붙인다, 불용어를 사용한다)
